# Vita 部署教程（完整版）

> 本文覆盖：**环境变量怎么填**、**数据库怎么装**、以及几种完整上线方式。  
> 仓库：https://github.com/YNT37/vita

---

## 0. 先选一种部署方式

| 方式 | 适合 | 数据库 | 访问入口 |
|------|------|--------|----------|
| **A. 本机开发** | 日常写代码 | SQLite（默认） | `localhost:3000` + `:5000` |
| **B. 云服务器 · venv + SQLite** | 考核演示、快速上线 | SQLite 文件 | `http://公网IP:3000` |
| **C. 云服务器 · venv + PostgreSQL** | 更稳、可多人写库 | 本机 Postgres | 同上 |
| **D. Docker Compose（SQLite）** | 不想装 Node/Python | 容器内 SQLite 卷 | `http://公网IP`（80） |
| **E. Docker Compose + Postgres** | 容器化正式一点 | Postgres 容器 | `http://公网IP`（80） |

**建议**：先 **B** 跑通；数据要稳再升到 **C** 或 **E**。  
**注意**：SQLite 与 PostgreSQL **数据不互通**，换库等于空库，需重新注册。

## 1. 配置文件总览

需要关心的文件（均已被 git 忽略，**不要提交密钥**）：

| 文件 | 从哪复制 | 作用 |
|------|----------|------|
| `backend/.env` | `backend/.env.example` | 后端：密钥、数据库、AI、CORS |
| `deploy/.env` | `deploy/.env.example` | 云服务器 venv：公网 IP、端口、venv 路径 |
| `frontend/.env.local` | 手写或本地示例 | 本机开发：后端地址 |

云服务器 venv 启动脚本会用 `deploy/.env` 里的 `PUBLIC_HOST` **构建前端**，不必再手写 `frontend/.env.local`。

## 2. `backend/.env` 逐项说明

```bash
cd backend
cp .env.example .env
vim .env
```

### 2.1 必改（上线）

| 变量 | 示例 | 说明 |
|------|------|------|
| `SECRET_KEY` | 随机长串 | Flask 会话等签名，**务必改掉** `change-me` |
| `JWT_SECRET` | 另一串随机 | JWT 签名，**与 SECRET_KEY 不要相同** |
| `JWT_EXPIRES_HOURS` | `24` | Token 有效小时数 |

生成随机串（Linux）：

```bash
openssl rand -hex 32
```

### 2.2 数据库 `DATABASE_URL`

| 场景 | 填法 |
|------|------|
| 本机 / 云服务器 SQLite（推荐入门） | **留空** → 使用 `backend/vita.db` |
| Docker Compose（SQLite 方案） | 一般由 compose 注入，见第 7 节 |
| 本机或同机 PostgreSQL | `postgresql://vita:密码@127.0.0.1:5432/vita` |
| Docker 里连 Postgres 服务名 `db` | `postgresql://vita:密码@db:5432/vita` |

说明：

- 旧写法 `postgres://` 也可，后端会自动改成 `postgresql://`。
- **不要把 Postgres 的 5432 对公网裸开放**；只让本机或 Docker 内网访问。
- 换 `DATABASE_URL` 后重启后端；表会由 SQLAlchemy `create_all` 自动建（空库）。

### 2.3 AI（可选，也可在网页「设置」里配）

| 变量 | 说明 |
|------|------|
| `AI_PROVIDER` | `openai`（兼容端点，含 DeepSeek）或 `anthropic` |
| `AI_API_KEY` | 密钥；也兼容旧名 `DEEPSEEK_API_KEY` |
| `AI_BASE_URL` | OpenAI 兼容基址，如 `https://api.deepseek.com` |
| `AI_MODEL` | 如 `deepseek-chat` / `gpt-4o-mini` |

DeepSeek 示例：

```env
AI_PROVIDER=openai
AI_API_KEY=sk-你的key
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
```

不填 Key 时：对话/简报走**降级文案**，站点仍可登录、记账、提醒。

### 2.4 其它

| 变量 | 说明 |
|------|------|
| `CORS_ORIGINS` | 允许的前端来源。测试可用 `*`；正式可写 `http://IP:3000` 或域名 |
| `NOTIFY_DISPATCH_INTERVAL` | Server酱后台扫描秒数，`0` 关闭 |
| `NOTIFY_CRON_SECRET` | 可选，外部 cron 调推送接口时校验 |

## 3. `deploy/.env`（仅云服务器 venv 方式需要）

```bash
cp deploy/.env.example deploy/.env
vim deploy/.env
```

| 变量 | 说明 |
|------|------|
| `PUBLIC_HOST` | **公网 IP 或域名**（不要带 `http://`）。前端打包后浏览器用它访问后端 |
| `BACKEND_PORT` | 默认 `5000` |
| `FRONTEND_PORT` | 默认 `3000` |
| `VENV_DIR` | 留空 → 用 `backend/.venv`；已有虚拟环境则填绝对路径 |

安全组 / 防火墙需放行：`FRONTEND_PORT`、`BACKEND_PORT`（或前面用 Nginx 只开 80/443）。

## 4. 数据库部署详解

### 4.1 SQLite（零安装，默认）

1. `DATABASE_URL=` 留空。  
2. 启动后端后自动生成 `backend/vita.db`。  
3. **备份**：复制该文件即可。  
4. 适合：本机开发、单机演示、低并发。

### 4.2 在云服务器安装 PostgreSQL（Ubuntu 示例）

```bash
sudo apt update
sudo apt install -y postgresql postgresql-contrib
sudo systemctl enable --now postgresql

# 进入 postgres 用户
sudo -u postgres psql
```

在 `psql` 里执行（把 `你的强密码` 换成自己的）：

```sql
CREATE USER vita WITH PASSWORD '你的强密码';
CREATE DATABASE vita OWNER vita;
GRANT ALL PRIVILEGES ON DATABASE vita TO vita;
\q
```

若 Postgres 16+ 还需连库授权 schema（部分版本）：

```bash
sudo -u postgres psql -d vita -c "GRANT ALL ON SCHEMA public TO vita;"
```

确保本机可连（默认 `peer`/`scram`）：编辑 `pg_hba.conf` 让 `127.0.0.1` 用密码（一般 Ubuntu 包已允许），然后：

```bash
sudo systemctl restart postgresql
# 测试
psql "postgresql://vita:你的强密码@127.0.0.1:5432/vita" -c 'SELECT 1;'
```

在 `backend/.env` 写入：

```env
DATABASE_URL=postgresql://vita:你的强密码@127.0.0.1:5432/vita
```

重启后端后访问站点并注册账号 → 数据进 Postgres。

**备份：**

```bash
pg_dump "postgresql://vita:你的强密码@127.0.0.1:5432/vita" > vita-$(date +%F).sql
```

**恢复：**

```bash
psql "postgresql://vita:你的强密码@127.0.0.1:5432/vita" < vita-2026-07-13.sql
```

### 4.3 仅用 Docker 跑 Postgres（venv 后端仍跑在宿主机）

```bash
docker run -d --name vita-pg --restart unless-stopped \
  -e POSTGRES_USER=vita \
  -e POSTGRES_PASSWORD=你的强密码 \
  -e POSTGRES_DB=vita \
  -p 127.0.0.1:5432:5432 \
  -v vita_pgdata:/var/lib/postgresql/data \
  postgres:16
```

`backend/.env`：

```env
DATABASE_URL=postgresql://vita:你的强密码@127.0.0.1:5432/vita
```

## 5. 方式 A — 本机开发（venv + SQLite）

**前置**：Python 3.10+、Node 18+。

```bash
# 后端
cd backend
python3 -m venv .venv          # Windows: python -m venv .venv
source .venv/bin/activate      # Windows: .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
cp .env.example .env           # DATABASE_URL 留空即可
# 按需填 AI_API_KEY
python app.py                  # http://127.0.0.1:5000/api/health
```

另开终端：

```bash
cd frontend
echo 'NEXT_PUBLIC_API_BASE=http://localhost:5000' > .env.local
npm install
npm run dev                    # http://localhost:3000
```

本机改用 Postgres：先按 §4.2/4.3 建库，再在 `backend/.env` 填 `DATABASE_URL` 后重启 `python app.py`。

## 6. 方式 B — 云服务器 · venv + SQLite（推荐快速演示）

**前置**：公网 IP；`git`、`python3`、`python3-venv`、`nodejs`/`npm`；安全组放行 **3000、5000**。

```bash
git clone https://github.com/YNT37/vita.git
cd vita

cp deploy/.env.example deploy/.env
vim deploy/.env
# PUBLIC_HOST=你的公网IP

cp backend/.env.example backend/.env
vim backend/.env
# 必改 SECRET_KEY、JWT_SECRET
# DATABASE_URL 留空（SQLite）
# 可选 AI_* 

chmod +x deploy/*.sh
./deploy/venv-setup.sh
./deploy/venv-start.sh
```

验证：

```bash
curl -s http://127.0.0.1:5000/api/health
# 浏览器打开 http://公网IP:3000
```

更新代码：

```bash
cd ~/vita
./deploy/pull-and-up.sh
```

停止：

```bash
./deploy/venv-stop.sh
```

日志：`deploy/run/backend.log`、`deploy/run/frontend.log`。

## 7. 方式 C — 云服务器 · venv + PostgreSQL

1. 按 **§4.2** 或 **§4.3** 准备好 Postgres。  
2. 与方式 B 相同克隆与 `deploy/.env`。  
3. `backend/.env` 中设置：

```env
SECRET_KEY=...
JWT_SECRET=...
DATABASE_URL=postgresql://vita:你的强密码@127.0.0.1:5432/vita
CORS_ORIGINS=*
```

4. `./deploy/venv-setup.sh && ./deploy/venv-start.sh`  
5. 打开前端注册新用户（空库）。

从 SQLite **迁到** Postgres：没有自动迁移脚本；导出业务数据或重新录入。考核演示一般直接 Postgres 空库重来即可。

## 8. 方式 D — Docker Compose（SQLite + Caddy :80）

**前置**：Docker + Compose 插件；安全组放行 **80**。

```bash
git clone https://github.com/YNT37/vita.git
cd vita

cp backend/.env.example backend/.env
vim backend/.env
# 必改 SECRET_KEY、JWT_SECRET；AI 可选
# DATABASE_URL 可不管：compose 会覆盖为容器内 SQLite 卷

docker compose up -d --build
curl -s http://127.0.0.1/api/health
# 浏览器 http://公网IP/
```

更新：

```bash
git pull origin main
docker compose up -d --build
```

数据在 Docker 卷 `vita_data`。查看：`docker volume ls`。

## 9. 方式 E — Docker Compose + PostgreSQL

```bash
git clone https://github.com/YNT37/vita.git
cd vita

cp backend/.env.example backend/.env
vim backend/.env
```

`backend/.env` 关键与数据库相关的关键项：

```env
SECRET_KEY=请改成随机串
JWT_SECRET=请改成另一随机串
DATABASE_URL=postgresql://vita:请改密码@db:5432/vita
CORS_ORIGINS=*
```

数据库密码文件：

```bash
cp deploy/postgres.env.example deploy/postgres.env
vim deploy/postgres.env
# POSTGRES_PASSWORD=必须与 DATABASE_URL 里密码相同
```

启动：

```bash
docker compose -f docker-compose.postgres.yml --env-file deploy/postgres.env up -d --build
curl -s http://127.0.0.1/api/health
# 浏览器 http://公网IP/
```

更新：

```bash
git pull origin main
docker compose -f docker-compose.postgres.yml --env-file deploy/postgres.env up -d --build
```

## 10. 常见问题

### 前端能开、接口失败 / CORS
- venv：确认 `PUBLIC_HOST` 是**浏览器能访问的 IP**，改完后需重新 `./deploy/venv-start.sh`（会重新 build 前端）。  
- `CORS_ORIGINS` 测试阶段用 `*`。

### `curl /api/health` 通，浏览器注册失败
- 检查前端是否指向了错误后端（旧 build、IP 写错）。  
- 看 `deploy/run/backend.log`。

### SSH 几分钟断一次
- 多为空闲超时，与 Vita 无关。本机 `~/.ssh/config`：

```
Host *
  ServerAliveInterval 30
  ServerAliveCountMax 6
```

### 换数据库后登录不上
- 新库是空的，需重新注册；旧 SQLite 数据不会自动进来。

### 内存不够（小规格云主机）
- `npm run build` 很吃内存；可临时加 swap，或本机构建镜像再上传（进阶）。
