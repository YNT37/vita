# 环境变量（Docker 部署）

Docker 部署主要只改 **`backend/.env`**。  
返回 [部署索引](./README.md)。

```bash
cp backend/.env.example backend/.env
vim backend/.env
```

| 变量 | 是否必改 | 说明 |
|------|----------|------|
| `SECRET_KEY` | 上线必改 | `openssl rand -hex 32` |
| `JWT_SECRET` | 上线必改 | 另一串，勿与上相同 |
| `JWT_EXPIRES_HOURS` | 可选 | 默认 `24` |
| `DATABASE_URL` | 视库而定 | **SQLite：留空**（由 compose 注入卷路径）；Postgres：见下 |
| `AI_PROVIDER` | 可选 | `openai` / `anthropic` |
| `AI_API_KEY` | 可选 | 也可用网页「设置」 |
| `AI_BASE_URL` / `AI_MODEL` | 可选 | 如 DeepSeek 基址与模型名 |
| `CORS_ORIGINS` | 一般 `*` | 同域 Caddy 反代足够 |
| `NOTIFY_*` | 可选 | Server酱扫描；浏览器通知不依赖 |

### Postgres 时的 `DATABASE_URL`

```env
DATABASE_URL=postgresql://vita:密码@db:5432/vita
```

- 用户/库名/密码与 `deploy/postgres.env` 一致  
- 主机名必须是 **`db`**（compose 服务名）

### 不再需要的文件

| 文件 | 说明 |
|------|------|
| `deploy/.env` | 仅旧版 venv 脚本使用，**Docker 部署不用** |
| `frontend/.env.local` | Docker 构建时 `NEXT_PUBLIC_API_BASE=""`，走同域 `/api` |

模板注释见仓库内 `backend/.env.example`。
