# 部署常见问题

返回 [部署索引](./README.md)。

---

## 前端能开、接口失败 / CORS

- venv（B/C）：确认 `deploy/.env` 的 `PUBLIC_HOST` 是浏览器能访问的 IP；改完后必须重新 `./deploy/venv-start.sh`（会重新 build 前端）。
- `CORS_ORIGINS` 测试阶段用 `*`。

## `curl /api/health` 通，浏览器注册失败

- 前端是否指向错误后端（旧 build、IP 写错）。
- 查看 `deploy/run/backend.log` 或 `docker compose logs backend`。

## SSH 几分钟断一次

多为空闲超时，与 Vita 无关。本机 `~/.ssh/config`：

```
Host *
  ServerAliveInterval 30
  ServerAliveCountMax 6
```

## 换数据库后登录不上

新库是空的，需重新注册；旧 SQLite 数据不会自动进来。

## 内存不够（小规格云主机）

`npm run build` 很吃内存；可临时加 swap，或改用 [方式 D/E](./README.md) 在本机构建镜像（进阶）。

- [ ] `SECRET_KEY` / `JWT_SECRET` 已改成随机串  
- [ ] `DATABASE_URL` 与真实库一致（或故意留空用 SQLite）  
- [ ] 安全组端口已放行  
- [ ] 健康检查通过  
- [ ] 能注册、登录、记一笔
