# 环境变量说明

相关文件（均已被 git 忽略，**不要提交密钥**）：

| 文件 | 从哪复制 | 作用 |
|------|----------|------|
| `backend/.env` | `backend/.env.example` | 后端：密钥、数据库、AI、CORS |
| `deploy/.env` | `deploy/.env.example` | 云服务器 venv：公网 IP、端口、venv 路径 |
| `frontend/.env.local` | 手写 | 本机开发：后端地址 |
| `deploy/postgres.env` | `deploy/postgres.env.example` | 仅 Docker+Postgres |

云服务器 venv 启动脚本会用 `deploy/.env` 的 `PUBLIC_HOST` **构建前端**，一般不必再写 `frontend/.env.local`。

返回 [部署索引](./README.md)。

---

## 1. `backend/.env`

```bash
cd backend
cp .env.example .env
vim .env
```

### 1.1 必改（上线）

| 变量 | 说明 |
|------|------|
| `SECRET_KEY` | Flask 签名，**务必改掉** `change-me` |
| `JWT_SECRET` | JWT 签名，**不要与 SECRET_KEY 相同** |
| `JWT_EXPIRES_HOURS` | Token 有效小时，默认 `24` |

生成随机串：

```bash
openssl rand -hex 32
```

### 1.2 数据库 `DATABASE_URL`

| 场景 | 填法 |
|------|------|
| SQLite（入门） | **留空** → `backend/vita.db` |
| Docker Compose SQLite | 一般由 compose 覆盖，见 [方式 D](./D-Docker-SQLite.md) |
| 同机 PostgreSQL | `postgresql://vita:密码@127.0.0.1:5432/vita` |
| Docker 内连 `db` | `postgresql://vita:密码@db:5432/vita` |

详见 [数据库说明](./数据库-SQLite与PostgreSQL.md)。

### 1.3 AI（可选；也可在网页「设置」配置）

| 变量 | 说明 |
|------|------|
| `AI_PROVIDER` | `openai`（兼容端点含 DeepSeek）或 `anthropic` |
| `AI_API_KEY` | 密钥；兼容旧名 `DEEPSEEK_API_KEY` |
| `AI_BASE_URL` | 如 `https://api.deepseek.com` |
| `AI_MODEL` | 如 `deepseek-chat` |

DeepSeek 示例：

```env
AI_PROVIDER=openai
AI_API_KEY=sk-你的key
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
```

不填 Key：对话走降级文案，登录/记账/提醒仍可用。

### 1.4 其它

| 变量 | 说明 |
|------|------|
| `CORS_ORIGINS` | 测试可用 `*`；正式写前端来源 |
| `NOTIFY_DISPATCH_INTERVAL` | Server酱扫描秒数，`0` 关闭 |
| `NOTIFY_CRON_SECRET` | 可选，外部 cron 校验 |

## 2. `deploy/.env`（仅云服务器 venv：B / C）

```bash
cp deploy/.env.example deploy/.env
vim deploy/.env
```

| 变量 | 说明 |
|------|------|
| `PUBLIC_HOST` | 公网 IP 或域名（**不要** `http://`） |
| `BACKEND_PORT` | 默认 `5000` |
| `FRONTEND_PORT` | 默认 `3000` |
| `VENV_DIR` | 留空用 `backend/.venv`；可填已有 venv 绝对路径 |

安全组放行对应端口（或前面 Nginx 只开 80/443）。
