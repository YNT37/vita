# 数据库（Docker）

返回 [部署索引](./README.md)。

**SQLite 与 PostgreSQL 数据不互通**，换库 = 空库，需重新注册。

---

## 默认：SQLite

使用根目录 `docker-compose.yml`：

- 无需安装数据库  
- `backend/.env` 里 `DATABASE_URL` 可留空（compose 会设为容器内路径）  
- 数据在 Docker 卷 `vita_data`  

适合本地与考核演示。

---

## 可选：PostgreSQL

使用 `docker-compose.postgres.yml` + `deploy/postgres.env`。

1. `cp deploy/postgres.env.example deploy/postgres.env`，设置 `POSTGRES_PASSWORD`  
2. `backend/.env`：

```env
DATABASE_URL=postgresql://vita:同一密码@db:5432/vita
```

3. 启动命令见 [本地 Docker](./本地-Docker.md) / [云服务器 Docker](./云服务器-Docker.md) 中的「可选 PostgreSQL」小节  

- Postgres **不对公网暴露**端口，仅容器网络访问  
- 备份：`docker compose ... exec db pg_dump -U vita vita > backup.sql`

---

## 不要混用两套 compose

同一台机器不要同时 `docker compose up`（SQLite 版）和 `docker-compose.postgres.yml`，会抢 **80** 端口。先 `docker compose down` 再换另一套。
